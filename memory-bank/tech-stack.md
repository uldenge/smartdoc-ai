# SmartDoc AI — 技术栈选型

## 选型原则
- 最简单但最健壮（Vibe Coding 核心原则）
- 零基础新手友好，社区资源丰富
- 免费/开源优先，降低成本
- 适合 AI 结对编程开发

---

## 前端

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | **Next.js 14+ (App Router)** | React 生态最大、教程最多、AI 工具支持最好 |
| 语言 | **TypeScript** | 类型安全，AI 生成代码更准确 |
| UI 组件 | **shadcn/ui + Tailwind CSS** | 复制粘贴式组件、零依赖、美观现代 |
| 状态管理 | **Zustand** | 比 Redux 简单 10 倍，适合小项目 |
| 请求库 | **fetch + SWR** | 内置即可，SWR 处理缓存 |

## 后端

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | **Next.js API Routes** | 前后端一体，零配置部署，新手最友好 |
| 数据库 | **PostgreSQL (Supabase)** | 免费托管、自带认证、向量扩展 pgvector |
| ORM | **Drizzle ORM** | 类型安全、轻量、AI 友好 |
| 认证 | **Supabase Auth** | 免费开箱即用，支持邮箱/社交登录 |

## AI / RAG 核心

| 技术 | 选择 | 理由 |
|------|------|------|
| Embedding 模型 | **OpenAI text-embedding-3-small** | 便宜、快速、效果好 |
| 对话模型 | **用户自选（OpenAI / DeepSeek / 智谱）** | 灵活、成本可控 |
| 向量存储 | **Supabase pgvector** | 与数据库一体，无需额外服务 |
| 文本切分 | **LangChain TextSplitter** | 成熟的文档处理工具 |
| PDF 解析 | **pdf-parse** | 轻量纯 JS 方案 |

## 部署

| 技术 | 选择 | 理由 |
|------|------|------|
| 前端+后端 | **Vercel** | 免费额度大、Next.js 原生支持 |
| 数据库 | **Supabase 云** | 免费额度够 MVP 使用 |
| 文件存储 | **Supabase Storage** | 免费额度 1GB |

## 开发工具

| 工具 | 用途 |
|------|------|
| **Node.js 18+** | 运行时 |
| **pnpm** | 包管理器（比 npm 快） |
| **VS Code** | 编辑器 |
| **Git + GitHub** | 版本控制 |
| **牛马AI (Claude Code)** | AI 结对编程 |

## 费用估算（MVP 阶段）

| 服务 | 费用 |
|------|------|
| Supabase（数据库+认证+存储） | 免费 |
| Vercel（部署） | 免费 |
| 域名（可选） | ~$10/年 |
| AI API 调用（用户自带 Key） | 用户承担 |
| **总计** | **$0 ~ $10/年** |

---

## 为什么不选其他方案

| 方案 | 不选的原因 |
|------|-----------|
| Python FastAPI + React | 前后端分离增加复杂度，新手需要维护两个项目 |
| MongoDB | 不支持向量搜索，需要额外服务 |
| Pinecone / Weaviate | 额外的向量数据库服务，增加学习成本 |
| Docker 自部署 | 新手部署运维成本太高 |
